# User Research

## User Journey Mapping

- Identify the trigger that initiates the journey (what makes the user start?).
- Map each step as: **Action** (what they do), **Touchpoint** (where they do it), **Emotion** (how they feel).
- Mark pain points where users hesitate, make errors, or abandon the flow.
- Distinguish between the current-state journey and the desired future-state journey.
- Validate maps with real user data -- never rely solely on assumptions.

## Usability Heuristics (Nielsen)

Apply these during any interface evaluation:

1. Visibility of system status -- keep users informed about what is happening.
2. Match between system and real world -- use familiar language and concepts.
3. User control and freedom -- provide undo, cancel, and escape routes.
4. Consistency and standards -- follow platform conventions; do not reinvent patterns.
5. Error prevention -- constrain inputs and confirm destructive actions.
6. Recognition over recall -- make options and actions visible rather than memorized.
7. Flexibility and efficiency -- support shortcuts for expert users.
8. Aesthetic and minimal design -- remove information that is irrelevant or rarely needed.
9. Help users recover from errors -- explain what went wrong and how to fix it.
10. Help and documentation -- provide searchable, task-oriented guidance.

## Persona Creation

- Base personas on observed behavior patterns, not demographics alone.
- Each persona needs: name, role, goals, frustrations, and typical workflow.
- Limit to 3-5 primary personas -- more than that dilutes focus.
- Review and update personas quarterly as user data evolves.

## Accessibility Guidelines

- Meet WCAG 2.1 AA as the minimum standard.
- Ensure all interactive elements are keyboard-navigable and have visible focus states.
- Provide text alternatives for non-text content (images, icons, charts).
- Maintain a color contrast ratio of at least 4.5:1 for normal text.
- Test with screen readers (VoiceOver, NVDA) and verify logical reading order.
